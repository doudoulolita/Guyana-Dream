1-3 - HTML de base

Votre navigateur ne supporte pas HTML5!

Créer la page HTML

Pour faire ce tutoriel, vous devez connaître les bases du HTML.

  • Créez d'abord un dossier au nom de votre jeu. ex: guyana-dream.
  • Ouvrez un éditeur de texte (et non un logiciel de traitement de texte) et recopiez le code ci-dessous.
  • Enregistrez-le sous le nom "index.html" dans le dossier de votre jeu.
  • Ouvrez la page dans un navigateur web en double-cliquant-dessus ou en choisissant le navigateur via un clic droit. Seul le titre s'affiche, ainsi que le nom de l'onglet. Le canvas n'est pas visible.
  • Revenez à l'éditeur de texte ou réouvrez de nouveau le fichier HTML avec, faites un changement du titre et enregistrez la modification.
  • Le changement dans le navigateur ne se verra qu'après avoir réactualisé la page.

Fichier : index.html


	<!DOCTYPE html>
	<html>

		<head>	

			<title>Mon RPG perso - bases du HTML</title>
			<meta charset="utf-8" />

			<link rel="stylesheet" type="text/css" href="css/style.css" /><!-- lien vers la feuille de style -->

			<!--[if IE]><script type="text/javascript" src="js/excanvas.js"></script><![endif]-->

		</head>

		<body>

			<h1>Mon RPG perso - bases du HTML</h1><!-- titre de niveau 1 -->
	
			<canvas width="384" height="192">Votre navigateur ne supporte pas HTML5!</canvas><!-- emplacement du jeu -->

			<!-- On placera ici le lien vers le fichier en javascript -->

		</body>
	</html>

Explications du code HTML

Chaque partie est encadrée par des balises qui ont un rôle particulier. Il y a la plupart du temps une balise ouverte et une balise fermée, cette dernière étant reconnaissable par le slash après le chevron ouvrant.

Les commentaires, qui ne seront pas affichés dans le navigateur web, sont encadrés par des caractères particuliers. Ils s'afficheront en bleu dans l'éditeur de texte si ce dernier possède une coloration syntaxique (très souhaitable !). Ils vous aideront à comprendre le code. Placez les aussi dans le votre, afin de vous y retrouver plus tard.

Dans la partie "head", on trouve :

  • Un titre "title" qui s'affichera dans l'onglet du navigateur (mais pas dans la page)
  • Un indication pour le codage des caractères
  • le lien vers la feuille de style "style.css", placée dans un dossier nommé "css". Dédiée à la présentation, alors que le HTML correspond à la structure, elle nous permettra d'ajouter une bordure à notre canvas, emplacement où sera affiché ensuite notre jeu. Le code est donné plus bas.
  • On ajoute aussi en commentaire une ligne qui gère les problèmes posés par Internet Explorer.

Dans la partie "body", on place

  • le titre du jeu (balise "h1")
  • la balise "canvas", dans laquelle nous pourrons dessiner à l'aide du code javascript, que nous étudierons plus tard. On indique sa taille (width=largeur, height=hauteur). Si le navigateur ne sait pas le gérer, le message inscrit entre les balises d'ouverture et de fermeture s'affichera.
  • en dessous, en commentaire, est indiqué l'emplacement du lien vers notre futur fichier javascript.

La fin du code est classique. Ce sont les balises fermantes du body et du html.

Ajouter une bordure au canvas avec la feuille de style

  • Si vous ne l'avez pas déjà fait, créez un dossier nommé "css" dans le dossier de votre jeu.
  • Ouvrez un nouveau fichier dans votre éditeur de texte et recopiez le code ci-dessous.
  • Enregistrez le sous le nom "style.css" dans le dossier "css". En cas d'erreur, la bordure ne s'affichera pas sur la pahge html.
  • Vérifiez le code, en particulier les points-virgules, et le chemin de la feuille de style dans la page html.

Fichier : style.css, classé dans le dossier "css"


	canvas {
		border: solid; /* Ajoute une bordure au canvas */
	}

Explications de la feuille de style

On indique le nom de l'élément à qui on souhaite donner une présentation particulière, ici "canvas".

Entre parenthèse, on donne le code pour une bordure.

On donne le style "solid" à la bordure ("border"), après les 2 points. On pourrait préciser son épaisseur et sa couleur mais ici, nous simplifions.

Les commentaires sont encadrés différemment de ceux du code HTML
